✅ 修改完成

已成功統一 Upload Photos Folder 和 Add Photos 的處理方法，現在它們都使用相同的流程：

2. 修改詳情

Add Photos 修改前：

// 預先生成 dataURL const resizedDataURL = await resizeImage(file); const resizedFile = { name: file.name, size: file.size, type: file.type, lastModified: file.lastModified || Date.now(), dataURL: resizedDataURL }; newPhotos.push(resizedFile); // 使用 renderNewPhotosOnly() await renderNewPhotosOnly(newPhotos, lazyObserver);

Add Photos 修改後：

// 直接使用 File 對象，不預先生成 dataURL const newPhoto = { name: file.name, size: file.size, type: file.type, lastModified: file.lastModified || Date.now(), isNewlyAdded: true }; newPhotos.push(newPhoto); // 保持使用 renderNewPhotosOnly() await renderNewPhotosOnly(newPhotos, lazyObserver);

3. 統一後的處理流程

Upload Photos Folder 流程：

1. 用戶選擇照片文件夾 2. 創建 File 對象數組 3. 設置 allPhotos = imageFiles 4. 調用 renderPhotos(allPhotos, lazyObserver) - 在 renderPhotos() 中檢查 dataURL - 如果沒有 dataURL，調用 resizeImage() 生成 - 渲染所有照片到 DOM 5. 調用 autoSaveAllPhotosToIndexedDB() 保存到 IndexedDB 6. 調用 saveDataToStorage() 更新 localStorage

Add Photos 流程：

1. 用戶選擇額外照片 2. 檢查重複照片 3. 創建新照片對象（不預先生成 dataURL） 4. 添加到 allPhotos 數組 5. 調用 renderNewPhotosOnly(newPhotos, lazyObserver) - 在 renderNewPhotosOnly() 中檢查 dataURL - 如果沒有 dataURL，調用 resizeImage() 生成 - 只渲染新照片到 DOM 6. 調用 autoSaveAllPhotosToIndexedDB() 保存到 IndexedDB 7. 調用 saveDataToStorage() 更新 localStorage

4. 優勢

  • ✅ 統一的 dataURL 生成： 都在渲染函數中生成，避免預先生成
  • ✅ 統一的 File 對象處理： 都直接使用 File 對象
  • ✅ 統一的錯誤處理： 相同的 dataURL 驗證和錯誤處理
  • ✅ 保持渲染差異： Upload Photos Folder 渲染所有照片，Add Photos 只渲染新照片
  • ✅ 更好的性能： 避免重複的 dataURL 生成
  • ✅ 保持原有功能： 不改變用戶體驗

5. 測試建議

請測試以下功能以確保修改正確：

  1. Upload Photos Folder： 選擇文件夾，檢查照片是否正確顯示
  2. Add Photos： 添加額外照片，檢查是否與現有照片一起顯示
  3. 會話恢復： 重新載入頁面，檢查會話恢復模態框是否顯示
  4. 照片保存： 檢查照片是否正確保存到 IndexedDB
  5. 控制台日誌： 檢查是否有 "Session restore check:" 日誌
