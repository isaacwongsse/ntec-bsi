照片保存流程分析

🔍 問題分析

根據代碼分析，發現 "Upload Photos Folder" 和 "Add Photos" 使用相同的保存流程，但存在以下問題：

  • 照片 dataURL 損壞導致顯示問題
  • 會話恢復模態框不顯示
  • 照片短暫顯示後消失

1. Upload Photos Folder 流程

步驟 1: 用戶點擊 "Select Photo Folder" 按鈕
步驟 2: 調用 selectPhotoFolder() 函數
步驟 3: 使用 File System Access API 或 webkitdirectory 選擇文件夾
步驟 4: 過濾圖片文件並創建 allPhotos 數組
步驟 5: 調用 renderPhotos(allPhotos, lazyObserver) 渲染照片
步驟 6: 調用 autoSaveAllPhotosToIndexedDB() 保存到 IndexedDB
步驟 7: 調用 saveDataToStorage() 更新 localStorage

2. Add Photos 流程

步驟 1: 用戶點擊 "Add Photos" 按鈕
步驟 2: 觸發 addPhotosFileInput.click()
步驟 3: 文件選擇後觸發 change 事件
步驟 4: 檢查重複照片並處理新照片
步驟 5: 為新照片生成 dataURL（調用 resizeImage()）
步驟 6: 將新照片添加到 allPhotos 數組
步驟 7: 調用 renderNewPhotosOnly() 只渲染新照片
步驟 8: 調用 autoSaveAllPhotosToIndexedDB() 保存到 IndexedDB
步驟 9: 調用 saveDataToStorage() 更新 localStorage

3. 流程比較

項目 Upload Photos Folder Add Photos 狀態
照片處理 直接使用 File 對象 調用 resizeImage() 生成 dataURL ✅ 相同流程
保存到 IndexedDB autoSaveAllPhotosToIndexedDB() autoSaveAllPhotosToIndexedDB() ✅ 相同函數
更新 localStorage saveDataToStorage() saveDataToStorage() ✅ 相同函數
照片渲染 renderPhotos() - 渲染所有照片 renderNewPhotosOnly() - 只渲染新照片 ⚠️ 不同方法
dataURL 生成 在 renderPhotos() 中生成 在處理時預先生成 ⚠️ 不同時機

4. 關鍵函數分析

autoSaveAllPhotosToIndexedDB() 函數

async function autoSaveAllPhotosToIndexedDB() { try { // 檢查 both window.allPhotos 和 allPhotos const photosToCheck = window.allPhotos || allPhotos; if (!photosToCheck || photosToCheck.length === 0) { window.logger.log('No photos in allPhotos container to save'); return; } // 檢查哪些照片還沒有保存到 IndexedDB const existingPhotos = await loadPhotosFromIndexedDB(); const existingPhotoNames = new Set(existingPhotos.map(photo => photo.name)); // 找出需要保存的新照片（包括新添加的照片） const newPhotos = photosToCheck.filter(photo => { // 檢查是否是新添加的照片 if (photo.isNewlyAdded) { return true; } // 檢查是否在 IndexedDB 中不存在 return !existingPhotoNames.has(photo.name); }); if (newPhotos.length > 0) { await savePhotosToIndexedDB(newPhotos); } // 同時更新 localStorage 作為備份 await saveDataToStorage(); } catch (error) { window.logger.error('Error auto-saving photos to IndexedDB:', error); } }

5. 問題根源分析

問題 1: dataURL 損壞
照片的 dataURL 被截斷或損壞，導致圖片無法載入
問題 2: 會話恢復模態框不顯示
沒有看到 "Session restore check:" 日誌，說明模態框顯示邏輯沒有執行
問題 3: 照片短暫顯示後消失
照片在頁面初始化時短暫顯示約0.1秒，然後因為 dataURL 損壞而消失
問題 4: 保存流程不一致
Upload Photos Folder 和 Add Photos 在 dataURL 生成時機上不同

💡 解決方案

  1. 統一 dataURL 生成時機: 確保所有照片都在上傳時生成 dataURL
  2. 加強 dataURL 驗證: 檢查 dataURL 的有效性
  3. 添加錯誤處理: 為損壞的 dataURL 提供錯誤占位符
  4. 修復會話恢復模態框: 確保模態框正常顯示
  5. 統一保存流程: 確保兩種上傳方式使用相同的保存邏輯

6. 修復建議

建議 1: 在照片上傳時立即生成和驗證 dataURL
建議 2: 添加 dataURL 損壞檢測和修復機制
建議 3: 統一 Upload Photos Folder 和 Add Photos 的處理流程
建議 4: 修復會話恢復模態框的顯示邏輯
